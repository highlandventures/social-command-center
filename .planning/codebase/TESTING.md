## Mocking Strategy

### Mock Layers

1. **External APIs (fetch)**
   - Mock fetch at module level with `vi.fn()`
   - Simulate various response statuses, headers, delays
   - Use `@testing-library/msw` (Mock Service Worker) for integration tests

2. **tRPC**
   - Mock `@/lib/trpc-client` exports
   - Return mock query results with loading/error states
   - Verify query parameters

3. **Database (Prisma)**
   - Use separate test database (or in-memory SQLite with `@prisma/sqlite`)
   - Run migrations before tests
   - Clean up data after each test

4. **Environment Variables**
   - Set in `vitest.setup.ts`
   - Ensure encryption key is set to valid value
