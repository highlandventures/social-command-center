### 4. External Platform Adapters

**Purpose:** Translate between social platforms' APIs and internal data models.

#### X Platform Adapter

**Location:** `social-command-center/lib/x-adapter.js`

**Architecture:**
```
XPlatformAdapter (unified interface)
  ├─ XOfficialClient (OAuth 2.0 — official X API)
  │   └─ POST /tweets — writes only
  │   └─ GET /tweets/{id}/metrics — own-account analytics only
  └─ TwitterAPIioClient (third-party — TwitterAPI.io)
      └─ GET operations — reads only
```

**Routing Logic:**
- **WRITES:** Always route to `XOfficialClient` (official API)
- **READS:** Default to `TwitterAPIioClient` (cheaper), fallback to official if configured
- **Own-Account Analytics:** Always `XOfficialClient`

**Cost Tracking:**
- All requests logged via `prisma.aPICallLog.create()`
- Estimated costs embedded in adapter (e.g., $0.01 per write, $0.00015 per third-party read)

**Caching:**
- All read results cached in Redis (Vercel KV) with TTLs per type

#### Reddit Adapter

**Location:** `social-command-center/lib/reddit-adapter.js`

**Architecture:**
```
RedditAdapter (unified interface)
  └─ Official Reddit API (OAuth 2.0)
      ├─ POST /api/submit — publish posts
      ├─ POST /api/comment — comments
      └─ GET /r/{subreddit} — reads
```

**Cost & Rate Limits:**
- $0.24 per 1K calls (tracked in logs)
- Rate limit headers tracked in Redis

**Caching:**
- Subreddit posts, search results, user profiles cached in KV

## Request Flow (Example: Dashboard Analytics)

1. **Client (React Component):** `app/(dashboard)/page.jsx`
   - Calls `trpc.analytics.dashboard.useQuery({ range: '30d' })`

2. **tRPC Client:** `lib/trpc-client.js`
   - Routes to server-side tRPC procedure

3. **Server Procedure (tRPC Router)** — *Not shown in codebase snapshot*
   - Authenticates user (middleware)
   - Calls `prisma.post.findMany()` to fetch posts
   - Calls `prisma.postMetrics.findMany()` with aggregations
   - Calls sentiment analysis (likely Claude API call, cost tracked)
   - Returns structured `DashboardData` object

4. **Data Fetching via Adapters:**
   - For fresh follower counts → `XPlatformAdapter.getOwnAccountAnalytics()` or cached from `PostMetrics`
   - For sentiment drivers → AI call (Claude SDK)

5. **Response Back to Client:**
   - tRPC serializes response, sends to client
   - React component updates state with data
   - Charts (Recharts) render with real data

## Abstractions & Key Patterns

### 1. Platform Adapter Pattern

**Purpose:** Isolate platform-specific API logic from business logic.

**Example:** `XPlatformAdapter` wraps both official and third-party X APIs.
- Consumers call `adapter.searchTweets()` without knowing if it's hitting official or third-party
- Routing & fallback logic encapsulated

### 2. Encrypted Token Storage

**Pattern:** Tokens stored encrypted in database, decrypted on retrieval.

**Flow:**
```javascript
// On save
const encrypted = encrypt(oauthToken);
await prisma.account.create({ data: { accessToken: encrypted } });

// On use
const account = await prisma.account.findUnique(...);
const decrypted = decrypt(account.accessToken);
const adapter = new XPlatformAdapter(decrypted);
```

### 3. Cached Fetching

**Pattern:** Wrap all remote calls in `cachedFetch()` helper.

```javascript
async searchTweets(query) {
  return cachedFetch(`x:search:${query}`, CACHE_TTL.SEARCH_RESULTS, async () => {
    return this.thirdParty.searchTweets(query); // Only called if cache miss
  });
}
```

### 4. Cost Tracking via Middleware

**Pattern:** Every API call (X, Reddit, Claude) logged with estimated cost.

```javascript
// In adapter request()
await prisma.aPICallLog.create({
  data: {
    provider: 'x_official',
    endpoint: path,
    statusCode: res.status,
    responseTime: Date.now() - start,
    estimatedCost: 0.01,
  },
});
```

### 5. tRPC Procedures

**Pattern:** Define RPC-style procedures with type-safe inputs/outputs.

**Example:** (not shown in codebase, but referenced)
```javascript
// Pseudo-code for how it's used
router.analytics.dashboard = trpc.procedure
  .input(z.object({ range: z.enum(['7d', '30d', '90d']) }))
  .query(async ({ input }) => {
    const posts = await prisma.post.findMany(...);
    const metrics = await prisma.postMetrics.aggregate(...);
    return { totals, postScatter, heatmap, ... };
  });
```
