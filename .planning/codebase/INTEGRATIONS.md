## Social Media Platforms

### X (Twitter)

**API**: X Official API v2 (`https://api.x.com/2`)
- **Auth**: OAuth 2.0 (Bearer token)
- **Type**: Read/Write hybrid routing
- **Adapter**: `lib/x-adapter.js`

**Configuration**:
- `X_OFFICIAL_CLIENT_ID` — OAuth client ID
- `X_OFFICIAL_CLIENT_SECRET` — OAuth client secret
- `X_OFFICIAL_BEARER_TOKEN` — Bearer token for API access
- `X_READ_PROVIDER` — "third-party" (default) or "official"
- `X_READ_FALLBACK_TO_OFFICIAL` — auto-fallback to official API on third-party failure

**Capabilities**:
- Post publishing (via official API)
- Metrics collection (likes, retweets, replies, bookmarks, impressions, engagements)
- Mention detection
- Competitor tracking
- KOL activation detection
- Rate limit tracking in Vercel KV (per endpoint, 900s TTL)
- API call logging to `APICallLog` (estimated cost: $0.01/POST, $0.003/GET)

**Implementation**:
- Writes always route to Official X API
- Reads route to third-party by default (TwitterAPI.io) with optional fallback

## Authentication & Authorization

### NextAuth.js

**Service**: NextAuth.js 4.24 (open-source auth library)
- **Configuration**: `lib/auth.js`
- **Route**: `app/api/auth/[...nextauth]/route.js`

**Auth Strategy**: Credentials Provider (team-based)
- No email sending required (ideal for internal tools)
- Shared team access code via `TEAM_PASSWORD` environment variable
- Auto-creates users on first sign-in
- First user auto-promoted to ADMIN role

**Environment Variables**:
- `NEXTAUTH_URL` — Site URL (default: http://localhost:3000)
- `NEXTAUTH_SECRET` — Session secret (generate with `openssl rand -base64 32`)
- `TEAM_PASSWORD` — Shared access code for all users

**Features**:
- Session tokens stored in Postgres
- Last active timestamp tracking
- User roles (ADMIN, INTERNAL, AGENCY)
- User invitation workflow (`invitedBy` field)

## Platform Connection Flow

### X (Twitter) OAuth

**Flow**:
1. User clicks "Connect X" in account settings
2. `app/api/connect/x/route.js` — Initiates OAuth flow, redirects to X
3. X redirects back to `app/api/connect/x/callback/route.js`
4. Callback stores access/refresh tokens (encrypted) in `Account` table
5. Tokens refreshed automatically via `lib/token-refresh.js` before expiry

---

### Reddit OAuth

**Flow**:
1. User clicks "Connect Reddit" in account settings
2. `app/api/connect/reddit/route.js` — Initiates OAuth flow
3. Reddit redirects to `app/api/connect/reddit/callback/route.js`
4. Callback stores access token (encrypted) in `Account` table
5. Token refreshed automatically if `refreshToken` available
