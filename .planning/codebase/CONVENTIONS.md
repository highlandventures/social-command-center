### Caching Strategy
Vercel KV (Redis) is used for server-side caching with TTL-based expiration:

```jsx
export const CACHE_TTL = {
  USER_PROFILE: 86400,        // 24 hours
  POST_METRICS_FRESH: 900,    // 15 minutes
  SEARCH_RESULTS: 1800,       // 30 minutes
};

export async function cachedFetch(key, ttlSeconds, fetchFn) {
  const cached = await kv.get(key);
  if (cached) return cached;
  const fresh = await fetchFn();
  await kv.set(key, fresh, { ex: ttlSeconds });
  return fresh;
}
```

## Summary

The codebase follows a **server-driven architecture** with Next.js App Router, Prisma ORM, and tRPC for type-safe data fetching. Components are functional, use modern React hooks, and are styled with Tailwind. Platform integrations (X, Reddit) are abstracted into class-based adapters with rate limit tracking and cost logging. Security is prioritized through AES-256-GCM encryption for sensitive tokens and a singleton Prisma client pattern. Error handling is explicit but minimal, relying on error propagation. The development conventions emphasize clarity, modularity, and minimal magic.
