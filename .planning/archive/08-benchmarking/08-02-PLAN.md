**1. BenchmarkSelector component (components/BenchmarkSelector.jsx):**
'use client' component for triggering benchmark comparisons on a report.

Props: `{ reportId, coveragePeriod, onCompare }` where onCompare receives the comparison result.

Features:
- Two-mode toggle: "Period" and "Milestone" (pill-style toggle, similar to sub-tab pattern in competitor intel)
- Period mode: Row of buttons for WoW, MoM, QoQ, YoY. Highlight the selected one.
- Milestone mode: Dropdown of named milestones (fetched via trpc.milestones.list.useQuery()). Show "No milestones — create one" with link to /reports/milestones if empty.
- "Compare" button triggers trpc.reports.compareBenchmark.useMutation()
- Loading state: Show spinner on Compare button while mutation runs
- Error state: Show toast on error
- On success: Call onCompare(data) to pass results to parent

Style: Contained in a bg-surface-card rounded-lg border border-border p-4 card. Title "Benchmark Comparison" with subtitle text. Compact layout -- should not dominate the page.

**2. Wire BenchmarkSelector into report detail page (app/(dashboard)/reports/[id]/page.jsx):**
- Import BenchmarkSelector
- Add state: `const [benchmarkResult, setBenchmarkResult] = useState(null)`
- Render BenchmarkSelector below the report header (above ReportViewer), only for reports with coveragePeriod (enriched reports). Pass reportId, coveragePeriod, and onCompare={setBenchmarkResult}.
- When benchmarkResult is set:
  - If `benchmarkResult.noData` is true: Show an info banner "No data available for the comparison period. Try a different period or milestone."
  - Otherwise: Render a "Benchmark Results" section showing the KPIs with deltas using KPICard components (same grid layout as ReportViewer). Show the comparison period dates as a subtitle: "Compared against: {start} - {end}".
  - Add a "Clear comparison" button to reset benchmarkResult to null.
- Do NOT modify the existing ReportViewer rendering -- the benchmark results are an overlay/section above or below the existing report content.
