  <behavior>
    intel-context:
    - Returns condensed summary with Performance/Competitors/Audience sections when all 3 insight types exist
    - Returns partial summary when only some insight types exist
    - Returns fallback message "No intel data available yet..." when no insights exist
    - Truncates each section to ~150 chars to stay within ~500 token budget

    brand-voice:
    - Returns top N posts sorted by engagementRate descending for a given accountId
    - Returns empty array when accountId is null/undefined
    - Returns empty array when no published posts exist
    - Truncates post content to 200 chars
    - Only includes posts with engagementRate > 0

    draft-detector:
    - Detects messages containing "Here's a draft:" or similar patterns
    - Detects messages with ```draft or ```post code fences
    - Detects messages with numbered thread format (1. / Tweet 1 / Post 1)
    - Returns false for purely conversational messages
    - Detects "## Draft" or "## Post" markdown headers
  </behavior>
  <action>
    **Step 1: Install new dependencies**
    ```bash
    npm install ai @ai-sdk/anthropic react-markdown remark-gfm
    ```

    **Step 2: Add Prisma models**
    Add to prisma/schema.prisma:
    - CopilotThread model: id (cuid), userId (String), accountId (String?), title (String?), createdAt, updatedAt. Relations: user -> User (onDelete: Cascade), messages -> CopilotMessage[]. Index on [userId, updatedAt]. Map to "copilot_threads".
    - CopilotMessage model: id (cuid), threadId (String), role (String: "user"|"assistant"|"system"), content (String), metadata (Json?), createdAt. Relation: thread -> CopilotThread (onDelete: Cascade). Index on [threadId, createdAt]. Map to "copilot_messages".
    - Add `copilotThreads CopilotThread[]` relation to the User model (after aiInsights line).
    - Run `npx prisma db push` to apply schema changes.

    **Step 3: Add Sonnet cost constants to lib/api-costs.js**
    Add alongside existing Haiku constants:
    ```javascript
    CLAUDE_SONNET_INPUT_PER_TOKEN: 0.000003,   // $3.00 per 1M input tokens
    CLAUDE_SONNET_OUTPUT_PER_TOKEN: 0.000015,   // $15.00 per 1M output tokens
    ```

    **Step 4: Create lib/copilot/intel-context.js**
    Export `getCondensedIntelSummary()`:
    - Query AIInsight for latest non-dismissed entry of each type: PERFORMANCE_PATTERN, COMPETITOR_STRATEGY, AUDIENCE_QUESTION (3 parallel findFirst queries ordered by generatedAt desc).
    - For each found insight, extract a text summary from its JSON content (stringify if object), truncate to ~150 chars.
    - Format as: "Performance: {summary}\n\nCompetitors: {summary}\n\nAudience Questions: {summary}"
    - If no insights found at all, return: "No intel data available yet. Run the daily analysis cron to populate insights."
    - Import prisma from '@/lib/db'.

    **Step 5: Create lib/copilot/brand-voice.js**
    Export `getTopPostsForAccount(accountId, limit = 5)`:
    - If !accountId, return [].
    - Query Post where accountId matches, status='PUBLISHED', include metrics (orderBy fetchedAt desc, take 1), take 50 most recent by publishedAt.
    - Filter to posts with metrics[0]?.engagementRate > 0.
    - Sort by engagementRate descending, take top `limit`.
    - Map to { content: post.content.slice(0, 200), contentType: post.contentType, engagementRate: post.metrics[0].engagementRate }.
    - Import prisma from '@/lib/db'.

    **Step 6: Create lib/copilot/system-prompt.js**
    Export `buildSystemPrompt({ accountId, postMode, platform })`:
    - Call getCondensedIntelSummary() and getTopPostsForAccount(accountId) in parallel.
    - Assemble system prompt with sections: role description, current context (platform, postMode, accountId), intel summary, brand voice examples (numbered), guidelines for mode-aware content, prediction format, draft marking.
    - Keep total prompt under ~1000 tokens. Instructions section ~200 tokens max.
    - If brand examples empty, include note: "No brand voice examples available for this account."

    **Step 7: Create lib/copilot/draft-detector.js**
    Export `detectDraftContent(messageText)`:
    - Check for explicit draft markers: /(?:here'?s?\s+(?:a|the|your)\s+)?draft[:\s]/i, /```(?:draft|post|thread|tweet)/i, /##\s*(?:Draft|Post|Thread|Tweet)/i
    - Check for thread format: /(?:^|\n)\s*(?:\d+[.)]\s|Tweet \d|Post \d)/m
    - Return true if any pattern matches, false otherwise.

    **Step 8: Write tests (RED then GREEN)**
    Write tests FIRST for intel-context, brand-voice, and draft-detector. Mock prisma using vi.mock('@/lib/db'). Run tests to see them fail, then implement the modules.

    - intel-context.test.js: Mock AIInsight.findFirst to return/not return insights. Test all-present, partial, empty cases.
    - brand-voice.test.js: Mock Post.findMany to return posts with various engagement rates. Test sorting, filtering, truncation, empty cases.
    - draft-detector.test.js: Pure function, no mocks needed. Test each pattern and negative cases.
