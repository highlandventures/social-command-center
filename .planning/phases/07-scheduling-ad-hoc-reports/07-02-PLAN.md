**1. Prisma schema (`prisma/schema.prisma`):**
Add AdHocReport and AdHocReportMessage models per research spec:

```
model AdHocReport {
  id                String    @id @default(cuid())
  title             String?
  reportId          String?   // Link to generated Report
  status            String    @default("SCOPING") // SCOPING, GENERATING, READY, FAILED
  reportParams      Json?     // { dateStart, dateEnd, reportType, metricsScope, comparisonBaseline }
  snapshotIntervals Json?     // [24, 48] hours or null
  nextSnapshotAt    DateTime?
  createdById       String
  createdAt         DateTime  @default(now())
  updatedAt         DateTime  @updatedAt
  messages          AdHocReportMessage[]
  @@index([createdById, updatedAt])
  @@map("adhoc_reports")
}

model AdHocReportMessage {
  id          String   @id @default(cuid())
  adHocId     String
  role        String   // 'user' | 'assistant'
  content     String
  createdAt   DateTime @default(now())
  adHocReport AdHocReport @relation(fields: [adHocId], references: [id], onDelete: Cascade)
  @@index([adHocId, createdAt])
  @@map("adhoc_report_messages")
}
```

Run `npx prisma db push` after adding models.

**2. System prompt (`lib/adhoc/system-prompt.js`):**
Export `ADHOC_SYSTEM_PROMPT` constant per the research spec. Key elements:
- Instruct AI to ask clarifying questions about time range, metrics focus, comparison baseline, report type
- Limit to ONE round of clarifying questions (2-3 questions max)
- When user confirms, output JSON block: `{"action":"generate","params":{"title":"...","dateStart":"YYYY-MM-DD","dateEnd":"YYYY-MM-DD","reportType":"...","metricsScope":"...","comparisonBaseline":"..."}}`
- Frontend detects JSON block to trigger generation

**3. Param extractor (`lib/adhoc/param-extractor.js`):**
Export `extractReportParams(text)` that:
- Searches for JSON block in assistant message (regex for ```json ... ``` or raw JSON with `"action":"generate"`)
- Parses and validates the params object
- Returns `{ title, dateStart, dateEnd, reportType, metricsScope, comparisonBaseline }` or null if not found/invalid

**4. tRPC adhocReports router (`lib/routers/adhoc-reports.js`):**
Export `adhocReportsRouter` with procedures:
- `list` query: returns all AdHocReport for current user, ordered by updatedAt desc, including message count and status
- `get` query: returns single AdHocReport with all messages (for chat page)
- `create` mutation: creates new AdHocReport with status SCOPING, returns id
- `saveMessage` mutation: accepts { adHocId, role, content }, creates AdHocReportMessage. If role is 'assistant', check for extractReportParams -- if found, update AdHocReport with reportParams and trigger generation
- `generate` mutation: accepts { adHocId, reportParams }, sets status to GENERATING, calls generateEnrichedReport with params, creates Report record, links reportId, sets status READY. On error: sets status FAILED.
- `rerun` mutation: accepts { adHocId }, reads stored reportParams, calls generateEnrichedReport again, creates new Report, updates reportId and status
- `configureSnapshots` mutation: accepts { adHocId, intervals } (array of hours like [24, 48]), sets snapshotIntervals and computes nextSnapshotAt as now + min(intervals) hours

Register `adhocReportsRouter` in `lib/routers/app.js` under `adhocReports` namespace.

**5. Streaming chat endpoint (`app/api/adhoc-report/chat/route.js`):**
Mirror the copilot chat pattern exactly:
- Auth via getServerSession
- Accept { messages, adHocId } from body
- If no adHocId, create new AdHocReport (status: SCOPING, createdById: session.user.id)
- Save latest user message to AdHocReportMessage
- Call streamText with ADHOC_SYSTEM_PROMPT, claude-sonnet-4-20250514, maxTokens: 2048
- In onFinish: save assistant message to AdHocReportMessage, log cost to APICallLog
- Check for extractReportParams in assistant response -- if found, update AdHocReport with params (status remains SCOPING until user confirms via frontend)
- Return result.toDataStreamResponse()
- Include adHocId in response headers: `result.toDataStreamResponse({ headers: { 'X-AdHoc-Id': activeAdHocId } })`

**6. Snapshot processing in cron (`app/api/cron/run-schedules/route.js`):**

- Query: `prisma.adHocReport.findMany({ where: { nextSnapshotAt: { lte: now }, status: 'READY' }, take: 3 })`
- For each: regenerate report using stored reportParams, advance nextSnapshotAt to next interval (or clear if all intervals exhausted)
- This piggybacks on the same cron route per research recommendation
