Add a MILESTONES section after the AD HOC REPORTS section:

```prisma
// ============================================================
// MILESTONES (Benchmarking)
// ============================================================

model Milestone {
  id          String   @id @default(cuid())
  name        String
  description String?
  startDate   DateTime
  endDate     DateTime
  createdById String
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@index([startDate, endDate])
  @@map("milestones")
}
```

Note: Use `DateTime` (not `@db.Date`) since this is SQLite via Prisma (dev.db). SQLite does not support `@db.Date`.

**1. Benchmark comparison module (lib/benchmark-compare.js):**
Create a lightweight comparison function that reuses existing report-engine infrastructure.

Export `computeBenchmarkDeltas(coveragePeriod, benchmarkPeriod)`:
- Takes two period objects: `{ start: Date, end: Date }`
- Fetches data for both periods using the same data-fetching pattern as `generateEnrichedReport` (look at how it calls prisma.post.findMany, prisma.accountMetrics.findFirst, prisma.listeningHit.findMany inside report-engine.js -- extract or reuse that pattern)
- Calls `calculateKPIs()` for both periods
- Calls `calculateDelta()` for each numeric KPI pair
- Returns `{ kpis: currentKPIsWithDeltas, benchmarkPeriod: { start, end } }`
- Handle edge case: if previous period has zero/null data for all KPIs, return a `noData: true` flag so UI can show "No data for comparison period" instead of misleading flat arrows

Export `resolveComparisonPeriod(coverageStart, coverageEnd, comparisonType)`:
- Maps comparison type to date range:
  - 'WoW': getPreviousPeriod with 7-day offset
  - 'MoM': getPreviousPeriod with month offset
  - 'QoQ': getPreviousPeriod with quarter offset
  - 'YoY': getPreviousPeriod with year offset
- For standard cadence types, use `getPreviousPeriod(new Date(coverageStart), new Date(coverageEnd))` which already computes same-duration previous period. The comparison type labels (WoW/MoM etc.) are suggestive for the UI -- the actual computation always mirrors the report's coverage duration.
- Returns `{ start: Date, end: Date }` or null if invalid

Import `calculateKPIs`, `calculateDelta`, `getPreviousPeriod` from `./report-engine.js`.
Import prisma from `@/lib/db`.
