1. In prisma/schema.prisma, add a CompetitorPost model in the COMPETITORS section after CompetitorMetrics:

```prisma
model CompetitorPost {
  id                String   @id @default(cuid())
  competitorId      String
  platform          Platform
  platformPostId    String
  content           String
  contentType       String   @default("POST")   // POST, THREAD, ARTICLE
  authorUsername    String
  postedAt          DateTime
  fetchedAt         DateTime @default(now())

  // Engagement metrics at time of fetch
  likes             Int      @default(0)
  retweets          Int      @default(0)
  replies           Int      @default(0)
  quotes            Int      @default(0)
  impressions       Int      @default(0)
  engagementRate    Float    @default(0)

  competitor        Competitor @relation(fields: [competitorId], references: [id], onDelete: Cascade)

  @@unique([competitorId, platformPostId])
  @@index([competitorId, postedAt])
  @@map("competitor_posts")
}
```

Add `posts CompetitorPost[]` relation to the Competitor model (alongside existing `metrics` relation).

Add COMPETITOR_STRATEGY to the InsightType enum (after PERFORMANCE_PATTERN).

2. In app/api/cron/poll-competitors/route.js, inside the tweet iteration loop (around line 84-93), after extracting engagement metrics from each tweet, add an upsert to store the post:

```javascript
// Inside the `for (const tweet of tweets)` loop, after computing `eng`:
const tweetText = tweet.text || tweet.full_text || '';
const tweetId = tweet.id || tweet.id_str || '';
const postedAt = tweet.created_at ? new Date(tweet.created_at) : new Date();
const impressionsCount = pm.impression_count || pm.impressions || 0;
const followersForRate = followersX || 1;

if (tweetId && tweetText) {
  await prisma.competitorPost.upsert({
    where: {
      competitorId_platformPostId: {
        competitorId: comp.id,
        platformPostId: String(tweetId),
      },
    },
    update: {
      likes: pm.like_count || pm.likes || 0,
      retweets: pm.retweet_count || pm.retweets || 0,
      replies: pm.reply_count || pm.replies || 0,
      quotes: pm.quote_count || pm.quotes || 0,
      impressions: impressionsCount,
      engagementRate: followersForRate > 0 ? (eng / followersForRate) * 100 : 0,
    },
    create: {
      competitorId: comp.id,
      platform: 'X',
      platformPostId: String(tweetId),
      content: tweetText,
      contentType: 'POST',
      authorUsername: acct.username,
      postedAt,
      likes: pm.like_count || pm.likes || 0,
      retweets: pm.retweet_count || pm.retweets || 0,
      replies: pm.reply_count || pm.replies || 0,
      quotes: pm.quote_count || pm.quotes || 0,
      impressions: impressionsCount,
      engagementRate: followersForRate > 0 ? (eng / followersForRate) * 100 : 0,
    },
  });
}
```

Place this upsert inside the existing `for (const tweet of tweets)` loop, right after the engagement computation (line ~91). The `followersX` variable is already available in scope from line 53. Use `acct.username` for authorUsername (available from the for-of loop at line 58).

<task type="auto">
  <name>Task 2: Create competitorIntel tRPC router with themes, formatAnalysis, and strategyCards procedures</name>
  <files>lib/routers/competitor-intel.js, lib/routers/app.js</files>
  <action>
Create lib/routers/competitor-intel.js following the performanceIntel router pattern:

**Import pattern:**
```javascript
import { z } from 'zod';
import { router, protectedProcedure } from '../trpc';
import { generateInsight } from '../ai';
```

**Shared input:** Accept `{ days: z.number().min(1).max(90).default(30), competitorId: z.string().optional() }` for filtering by time range and optionally by competitor.

**Procedure 1: `themes`** (addresses COMP-02)
- Query CompetitorPost records within the date range (filtered by competitorId if provided)
- Extract pure helper function `computeCompetitorThemes(posts)` that:
  - Uses bigram/trigram extraction from post content (similar to computeTopicSignals in performance-intel.js but applied to competitor posts)
  - Groups by phrase, counts occurrences across posts, computes average engagement per phrase
  - Returns top 20 themes sorted by occurrence count
- Return `{ themes: [{ phrase, occurrences, avgEngRate, postCount }], totalPosts, competitors: [names] }`

**Procedure 2: `formatAnalysis`** (addresses COMP-03)
- Query CompetitorPost records within date range
- Extract pure helper function `computeCompetitorFormats(posts)` that:
  - Groups posts by contentType
  - Computes per-format: count, avgEngRate, avgLikes, avgRetweets, topPost (highest engagement)
  - Returns sorted by avgEngRate descending
- Return `{ formats: [{ format, postCount, avgEngRate, avgLikes, avgRetweets, topPost: { content (truncated 120 chars), engagementRate } }] }`

**Procedure 3: `strategyCards`** (addresses COMP-04)
- Query all competitors with their CompetitorPost records and latest CompetitorMetrics
- Also query Figure's own posts (via Post model with status PUBLISHED) for comparison
- For each competitor, compute:
  - postingCadence: posts per day (total posts / days in range)
  - topThemes: top 3 themes from their posts (reuse computeCompetitorThemes filtered to that competitor)
  - avgEngRate from their posts
  - followersX from latest CompetitorMetrics
- Build a context object with all competitor stats + Figure's stats
- Call `generateInsight('competitor_strategy', context, { model: 'claude-3-5-haiku-20241022', maxTokens: 1500, systemPrompt })` where systemPrompt instructs:
  "You are a competitive intelligence analyst for a social media team. Analyze competitor posting strategies and compare to our performance. Return a JSON array of strategy cards, one per competitor. Each card: { competitorName, postingCadence (string like '2.3 posts/day'), topThemes (array of 3 strings), formatMix (string), engagementBenchmark (string comparing to our avg), keyInsight (string, 1-2 sentences of strategic takeaway) }."
- Cache results using AIInsight with insightType COMPETITOR_STRATEGY, check for non-dismissed insights from last 24h before regenerating (same caching pattern as performanceIntel.insightCards)
- Return the array of strategy card objects

**Export:** `export const competitorIntelRouter = router({ themes, formatAnalysis, strategyCards });`
Also export helper functions for testability: `export { computeCompetitorThemes, computeCompetitorFormats }`

**Register in app.js:**
- Import: `import { competitorIntelRouter } from './competitor-intel';`
- Add to router: `competitorIntel: competitorIntelRouter,`
