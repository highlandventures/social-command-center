# Phase 6: Export + Distribution - Context

**Gathered:** 2026-03-15
**Status:** Ready for planning

<domain>
## Phase Boundary

PDF export and email distribution for enriched reports. Users can export any report as a branded PDF and email reports to configurable recipients with a visual summary inline. This phase builds on the enriched report JSON from Phase 5 (KPIs, charts, executive summary, sentiment themes, recommendations). Slack integration is explicitly excluded from this phase scope.

</domain>

<decisions>
## Implementation Decisions

### PDF Generation
- Use `@react-pdf/renderer` to generate PDFs server-side from enriched report JSON
- Mirror the web ReportViewer layout: KPI bar → executive summary → charts → sentiment themes → top content → recommendations — paginated for print with automatic page breaks between major sections
- Chart images already stored as Vercel Blob URLs from Phase 5 — PDF references them directly via `@react-pdf/renderer`'s `<Image>` component
- Generated PDF uploaded to Vercel Blob for persistent URL, linked on report detail page
- Increment `downloads` counter on the Report model when PDF is downloaded

### PDF Branding — Figure Brand
- **Branded header on every page:** Figure wordmark logo (top-left), report title + date range (top-right), Figure Purple (#5B56F5) accent bar across the top
- **Branded footer on every page:** "Generated by Social Command" (left), page number "Page X of Y" (right), light gray divider line above
- **Color palette from Figure brand guide:**
  - Figure Purple: #5B56F5 (primary accent, header bar, section dividers)
  - Black: #1A1B22 (primary text)
  - Bone: #F2F2F2 (KPI card backgrounds, alternating section backgrounds)
  - Concrete: #9CADBB (secondary text, labels, muted content)
  - Sienna: #EF8260 (negative delta indicators)
  - Slate Blue: #B2C5D5 (borders, dividers)
- **Typography:**
  - Sharp Grotesk SmBold — report title, section headings (registered as custom font in @react-pdf/renderer)
  - Plus Jakarta Sans SemiBold — KPI values, emphasis text
  - Plus Jakarta Sans Regular (fallback to Helvetica) — body text, descriptions
- **Logo assets stored at:** `public/brand/logos/` (Figure_Logo_Purple.png for PDF header)
- **Font files stored at:** `public/brand/fonts/` (SharpGroteskSmBold21.otf, PlusJakartaSans-SemiBold.ttf, PlusJakartaSans-Bold.ttf)
- Paper size: US Letter (8.5" x 11"), portrait orientation
- No cover page — report starts directly with branded header + KPI bar on page 1

### Email Distribution
- Use `@react-email/components` for email template + `nodemailer` for SMTP delivery
- **Email content: Summary + link + PDF attachment** — KPI highlights and executive summary rendered inline in the email body, "View Full Report" button linking to the app, PDF attached for offline reading
- **Styled HTML email template** — branded with Figure Purple accent, clean section headers, responsive layout. Professional feel, not plain text
- Email header: Figure wordmark + report title + date range
- Email body sections: KPI summary table (5 metrics with deltas), executive summary paragraph, "View Full Report" CTA button
- Chart images NOT inlined in email (too unreliable across clients) — recipients get the PDF attachment for full visuals
- SMTP provider configuration via environment variables (SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS, SMTP_FROM)

### Recipient Configuration
- Simple saved defaults approach: one default recipient list stored in app settings, overridable per send
- "Email Report" action on report detail page opens a modal with pre-filled recipients from defaults
- User can add/remove email addresses before sending
- Default recipient list managed in a Settings section (or inline on first send)
- Recipients stored as JSON array of email strings — no separate User/Recipient model needed for v1.1

### Delivery Tracking
- Delivery log table on each report's detail page showing: date sent, recipients, status (sent/failed), channel (email/PDF download)
- New `ReportDelivery` model in Prisma: id, reportId, channel (EMAIL/PDF_DOWNLOAD), recipients (JSON), status (SENT/FAILED), error (nullable), createdAt
- Report detail page shows delivery history below the report content
- No dedicated distribution tab across all reports — keep it scoped to per-report view

### UI Integration
- Report detail page action toolbar: "Export PDF" button (generates + downloads) and "Email Report" button (opens recipient modal)
- These buttons were placeholder/disabled in Phase 5 — now activate them
- PDF generation shows loading state while rendering (can take a few seconds for chart image fetching)
- Email modal: recipient input field (comma-separated or tag-style), subject line (auto-filled from report title), send button with loading state
- Success/error toast notifications for both export and email actions

### Claude's Discretion
- PDF page break logic — where exactly to split between sections
- Email template responsive breakpoints and exact spacing
- Loading/progress indicators during PDF generation
- Error handling UX for SMTP failures
- Exact modal layout for email recipient input
- Whether to show a "Generating PDF..." progress bar or simple spinner

</decisions>

<specifics>
## Specific Ideas

- PDF should feel like a premium deliverable — something an exec would forward to their board without embarrassment
- The Figure Purple accent bar at the top of each PDF page is a subtle but effective brand touch
- Email summary should answer "how are we doing?" in the inbox preview — KPI numbers visible without opening
- PDF attachment means the report travels well — recipients can save, forward, print without needing app access
- Keep recipient management dead simple for now — a comma-separated list is fine, fancy tag inputs can come later

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- `lib/report-engine.js`: `generateEnrichedReport()` produces canonical JSON with kpis[], charts[], executiveSummary, sentimentThemes, recommendations, topContent — this is the data source for both PDF and email
- `lib/chart-renderer.js`: Chart images already at Vercel Blob URLs (1200x600px PNGs) — PDF and email reference these directly
- `components/ReportViewer.jsx`: Web layout to mirror in PDF — KPI stats row, executive summary, chart images, sentiment themes, top content, recommendations
- `lib/routers/reports.js`: tRPC router to extend with `exportPdf` and `emailReport` procedures
- `components/ui.jsx`: KPICard, SectionTitle, DeltaArrow components — reference for PDF styling
- `lib/ai.js`: Not directly needed for Phase 6 (no AI generation), but report content already contains AI-generated text

### Brand Assets (newly added)
- `public/brand/logos/Figure_Logo_Purple.png` — purple wordmark on transparent bg (for PDF header)
- `public/brand/logos/Figure_Logo_White.png` — white wordmark (for email header on purple bg)
- `public/brand/logos/Figure_F_Logo_PurpleFill.png` — F icon on purple square (for email favicon/social)
- `public/brand/fonts/SharpGroteskSmBold21.otf` — heading font
- `public/brand/fonts/PlusJakartaSans-SemiBold.ttf` — body emphasis font
- `public/brand/fonts/PlusJakartaSans-Bold.ttf` — body bold font

### Established Patterns
- Vercel Blob upload pattern from `lib/chart-renderer.js`: `put(path, buffer, { access: 'public' })` — reuse for PDF storage
- tRPC protectedProcedure for authenticated endpoints
- Report model stores content as JSON blob — no schema changes needed for report content itself
- Toast notifications via existing toast system in the app

### Integration Points
- Report detail page (needs "Export PDF" and "Email Report" buttons activated)
- Prisma schema: new `ReportDelivery` model for delivery tracking
- `lib/routers/app.js`: register new distribution router if separate from reports router
- Environment variables: SMTP_HOST, SMTP_PORT, SMTP_USER, SMTP_PASS, SMTP_FROM
- `package.json`: add `@react-pdf/renderer` and `@react-email/components` dependencies

</code_context>
