3. Add `ReportDelivery` model to `prisma/schema.prisma` after the Report model:
   ```prisma
   model ReportDelivery {
     id         String   @id @default(cuid())
     reportId   String
     channel    String   // 'EMAIL' or 'PDF_DOWNLOAD'
     recipients Json     @default("[]") // JSON array of email strings
     status     String   // 'SENT' or 'FAILED'
     error      String?
     createdAt  DateTime @default(now())
     report     Report   @relation(fields: [reportId], references: [id], onDelete: Cascade)
     @@map("report_deliveries")
   }
   ```
   Also add `deliveries ReportDelivery[]` relation to the Report model.
   Run `npx prisma db push` to sync schema.

4. Create `lib/pdf-renderer.js` with:
   - Font registration at module scope using `path.join(process.cwd(), 'public/brand/fonts/...')` for Sharp Grotesk and Plus Jakarta Sans. Register Helvetica fallback via `Font.registerHyphenationCallback(word => [word])`.
   - BRAND color constants: purple #5B56F5, black #1A1B22, bone #F2F2F2, concrete #9CADBB, sienna #EF8260, slateBlue #B2C5D5
   - StyleSheet with: page (US Letter, paddings for header/footer space), headerBar (absolute position, 4px purple bar across top), header (absolute, logo left + title/date right), footer (absolute bottom, "Generated by Social Command" left, page number right with render callback), kpiRow (flexDirection row, gap), kpiCard (bone background, rounded), sectionTitle (SharpGrotesk family, 14px), chartImage, recommendationRow, sentimentCard
   - `ReportPDF({ report, chartImages, logoBase64 })` component rendering:
     - Purple accent bar (fixed on every page)
     - Header: Figure logo (Image from logoBase64) left, report title + date range right (fixed)
     - Footer: "Generated by Social Command" left, "Page X of Y" right using render callback (fixed)
     - KPI row: horizontal cards with label (concrete, 7pt), value (PlusJakartaSans 600, 16pt), delta arrow text if delta exists
     - Executive Summary section with sectionTitle + body text (lineHeight 1.6)
     - Charts: each chart gets a `break` (page break before), section title, Image from chartImages[chart.id] or "Chart unavailable" fallback
     - Sentiment Themes: section title, then for each theme render theme name (bold) + bullet items
     - Top Content: section with top posts, title + metrics summary
     - Recommendations: section with break, each recommendation shows [priority] tag + text
   - `prefetchChartImages(charts)` async function: fetches each chart.imageUrl, converts to base64 data URI. Returns object keyed by chart.id. Gracefully returns null for failed fetches.
   - `renderReportPDF(report)` async helper that:
     a. Pre-fetches chart images via prefetchChartImages(report.content?.charts)
     b. Reads logo file as base64: `fs.readFileSync(path.join(process.cwd(), 'public/brand/logos/Figure_Logo_Purple.png'))` converted to data URI
     c. Calls `renderToBuffer(<ReportPDF report={report} chartImages={chartImages} logoBase64={logoBase64} />)`
     d. Returns the PDF buffer

   Use format-aware KPI value display: 'number' -> toLocaleString, 'percent' -> value + '%', 'delta' -> sign prefix, 'text' -> raw string. Mirror the KPICard format logic from components/ui.jsx.

<task type="auto">
  <name>Task 2: PDF API route and Export PDF button on report detail page</name>
  <files>app/api/pdf/[id]/route.js, app/(dashboard)/reports/[id]/page.jsx</files>
  <action>
1. Create `app/api/pdf/[id]/route.js`:
   - Export `maxDuration = 30` for Vercel timeout
   - GET handler that:
     a. Gets report ID from params
     b. Fetches report from prisma (findUnique by id), returns 404 if not found
     c. Calls `renderReportPDF(report)` from `@/lib/pdf-renderer`
     d. Uploads PDF buffer to Vercel Blob: `put(\`reports/pdf/\${report.id}-\${Date.now()}.pdf\`, buffer, { access: 'public', contentType: 'application/pdf' })`
     e. Creates ReportDelivery record: `{ reportId: report.id, channel: 'PDF_DOWNLOAD', recipients: [], status: 'SENT' }`
     f. Increments report.downloads: `prisma.report.update({ where: { id: report.id }, data: { downloads: { increment: 1 } } })`
     g. Returns JSON `{ url: blob.url }` with status 200
   - Wrap in try/catch: on error, return 500 with `{ error: 'PDF generation failed' }`
   - Import prisma from `@/lib/db`, put from `@vercel/blob`, renderReportPDF from `@/lib/pdf-renderer`

2. Update `app/(dashboard)/reports/[id]/page.jsx`:
   - Add "Export PDF" button in the header area next to the title/status section

   - On click: set loading true, fetch `/api/pdf/${id}`, parse JSON response, open `result.url` in new tab (or create a temporary anchor to trigger download), set loading false
   - Button styling: `bg-gray-900 text-white text-sm font-medium rounded-lg px-4 py-2 hover:bg-gray-800 disabled:opacity-50` with a download icon or text "Export PDF" / "Generating..." when loading
   - Add an "Email Report" button placeholder next to it (disabled, will be activated in Plan 02): `bg-white border border-gray-300 text-gray-700 text-sm font-medium rounded-lg px-4 py-2 opacity-50 cursor-not-allowed` with text "Email Report"
   - Show toast on success ("PDF ready -- downloading") and error ("PDF generation failed") using a simple inline toast or alert. If there's an existing toast system use it; otherwise a temporary div.
   - Handle both success and network error cases in the fetch
