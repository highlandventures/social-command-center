<objective>
Close the PRD §12 metrics-gap trio before Phase 18 exposes `social.get_post_metrics` over MCP: (1) add `quotes` to `PostMetrics` and stop discarding the `quoteCount` value that `poll-metrics` already reads from TwitterAPI.io; (2) replace simple-mean engagement-rate aggregation in `report-engine.js` with an impression-weighted average so a high-impression post doesn't get averaged out by a zero-impression one; (3) extend the poll-metrics accrual window per the PRD so late-arriving engagements aren't dropped. Exposing deflated numbers through MCP would erode team trust on day one (PRD §Phase 1 "Fixes bundled").

Output: one schema column, two cron-route fixes, one library helper rewrite, targeted unit tests.
</objective>
