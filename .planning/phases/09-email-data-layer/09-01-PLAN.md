**email-lists.js** — Follow apps/social/lib/routers/schedules.js pattern exactly:

```javascript
import { z } from 'zod';
import { router, protectedProcedure } from '../trpc';
```

Procedures:
- `list`: query, findMany ordered by createdAt desc, include `_count: { select: { subscribers: true } }`
- `create`: mutation, input z.object({ name: z.string().min(1).max(100), description: z.string().max(500).optional() }), create with createdById: ctx.user.id
- `update`: mutation, input z.object({ id: z.string(), name: z.string().min(1).max(100).optional(), description: z.string().max(500).optional() }), update where id
- `delete`: mutation, input z.object({ id: z.string() }), delete where id

**email-subscribers.js** — Follow apps/social/lib/routers/posts.js cursor pagination pattern:

Procedures:
- `list`: query with input { listId (required), status (optional SubscriberStatus enum), search (optional string), limit (default 50, max 100), cursor (optional) }. Build where clause: always filter by listId. If status provided, add status filter. If search provided, add OR clause for email/firstName/lastName contains (mode: insensitive). Use cursor-based pagination pattern from posts.js (take limit+1, pop last for nextCursor). Return { items, nextCursor }.
- `create`: mutation, input { listId, email (z.string().email()), firstName (optional), lastName (optional) }. Normalize email to lowercase. Create with status: 'ACTIVE'.
- `importCSV`: mutation, input { listId: z.string(), subscribers: z.array(z.object({ email: z.string().email(), firstName: z.string().nullish(), lastName: z.string().nullish() })).max(10000) }. Use prisma.emailSubscriber.createMany with data mapped (email lowercase, status ACTIVE) and skipDuplicates: true. Return { imported: result.count, total: input.subscribers.length }.
- `updateStatus`: mutation, input { id: z.string(), status: z.enum(['ACTIVE', 'UNSUBSCRIBED', 'BOUNCED', 'COMPLAINED']) }. Update where id.
- `delete`: mutation, input { id: z.string() }. Delete where id.
