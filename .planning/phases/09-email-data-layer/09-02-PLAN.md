Build the subscriber management page for a specific email list. This is the most feature-rich page in Phase 9.

1. 'use client' directive
2. Import trpc, useState, useParams from next/navigation, Link
3. Fetch list details: `trpc.emailLists.list.useQuery()` and find by params.id (or create a `getById` — simpler to filter client-side from the list query since list count is small)
4. Fetch subscribers: `trpc.emailSubscribers.list.useQuery({ listId: params.id, status: statusFilter, search: searchQuery, limit: 50 })`

**Search and filter bar:**
- Text input for search (debounced 300ms — use useState + useEffect with setTimeout)
- Status dropdown filter: All, Active, Unsubscribed, Bounced, Complained
- "Import CSV" button (opens file dialog)
- "Add Subscriber" button (opens inline form)

**Subscriber table:**
- Columns: Email, First Name, Last Name, Status, Added (date), Actions (delete)
- Status shown as colored badge: Active (green), Unsubscribed (yellow), Bounced (red), Complained (red)
- Empty state when no subscribers: "No subscribers yet. Import a CSV or add manually."

**CSV Import flow:**
- "Import CSV" button triggers hidden `<input type="file" accept=".csv">`
- On file select, parse CSV client-side using FileReader (follow Pattern 3 from research exactly):
  - Read file as text
  - Split by newlines, filter empty
  - Parse header row (lowercase, trimmed) to find email/firstName/lastName column indexes
  - Map data rows to { email (lowercase), firstName, lastName } objects
  - Filter rows without valid email (must contain @)
  - Reject if no 'email' column found (show error toast)
- Show import preview: "{N} valid subscribers found. Import?"
- On confirm, call `trpc.emailSubscribers.importCSV.useMutation({ listId, subscribers })`
- Show result: "{imported} of {total} imported ({total - imported} duplicates skipped)"
- Invalidate subscriber list query on success

**Add subscriber form:**
- Inline expandable form (not a modal) below the toolbar
- Fields: email (required), first name (optional), last name (optional)
- Call `trpc.emailSubscribers.create.useMutation` on submit
- Invalidate list on success, clear form

**Pagination:**
- "Load more" button at bottom if nextCursor exists
- Use infinite query pattern or simple "load more" that fetches next page

**Back navigation:**
- Breadcrumb or back link: "Lists > {listName}"

Use consistent Tailwind styling with other dashboard pages.
