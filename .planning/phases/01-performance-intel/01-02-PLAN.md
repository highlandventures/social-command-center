**Step 2: Create `components/PerformanceIntelPanel.jsx`**

This is a `'use client'` component that renders the full Performance Intel panel for the composer sidebar.

Structure:
```
PerformanceIntelPanel
  |-- InsightCardsSection (top, most prominent)
  |-- TieredPostsSection (collapsible sections for top/avg/poor)
  |-- PatternCallouts (format + timing patterns)
```

**InsightCardsSection:**
- Call `trpc.performanceIntel.insightCards.useQuery({ range: '30d' })`
- Render each card as a compact card with colored left border based on category:
  - format: blue, timing: amber, topic: purple
- Show the `metric` prominently (e.g., "3.2x"), `title` as bold text, `body` as smaller description
- Show loading skeletons while fetching
- If no insights yet, show a muted "Analyzing your content..." state

**TieredPostsSection:**
- Call `trpc.performanceIntel.tieredPosts.useQuery({ range: '30d' })`
- Render 3 collapsible sections:
  - "Top Performers" (green left border, trophy icon via Unicode or emoji-free indicator)
  - "Average" (gray left border)
  - "Needs Improvement" (red left border)
- Each section header shows count: "Top Performers (5)"
- Default: Top section expanded, others collapsed
- Each post row shows:
  - Content snippet (first 80 chars, truncated)
  - PlatformBadge + contentType badge
  - Engagement rate as percentage
  - MiniSparkline (loaded via sparklineBatch)
- Call `trpc.performanceIntel.sparklineBatch.useQuery({ postIds })` where postIds are all post IDs from tieredPosts. Only call when tieredPosts data is loaded (use `enabled` option).

**PatternCallouts:**
- Call `trpc.performanceIntel.patternAnalysis.useQuery({ range: '30d' })`
- Render as compact callout cards:
  - Best format: "Threads get 2.1x more engagement than single posts"
  - Best time: "Posts on Tue 10am UTC get 45% higher engagement"
  - Top topic signal (if any): "Posts mentioning 'RWA' average 8.2% engagement"
- Use subtle background colors and small icons/symbols (no emojis)
- If data is insufficient (< 5 posts), show "Need more published posts for pattern analysis"

**Styling notes:**
- Follow existing Tailwind patterns from composer page
- Compact sizing (text-[10px] to text-xs) to fit sidebar width
- All sections use Skeleton loading states consistent with existing ui.jsx patterns
- Scrollable container with `overflow-y-auto` and `scrollbarWidth: 'thin'`

**Props:** `PerformanceIntelPanel` takes no props -- it manages its own data fetching via tRPC hooks.
