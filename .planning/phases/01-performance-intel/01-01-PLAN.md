**Step 2: Create `lib/routers/performance-intel.js`**

Create a new tRPC router file exporting `performanceIntelRouter` with these 4 procedures:

**Procedure 1: `performanceIntel.tieredPosts`**
- Input: `rangeInput` (same schema as analytics.js -- range enum + optional accountId)
- Query published posts with their latest PostMetrics (same pattern as `analytics.postPerformance`)
- Compute engagement rate per post: `engagements / impressions * 100`
- Rank posts by engagement rate and assign tiers using percentiles:
  - Top tier: >= 75th percentile engagement rate
  - Average tier: 25th-75th percentile
  - Poor tier: < 25th percentile
- Return `{ tiers: { top: Post[], average: Post[], poor: Post[] }, totalPosts: number, dateRange: { since, until } }` where each Post includes: `id, content (first 120 chars), contentType, platform, publishedAt, account.username, metrics: { impressions, engagements, engagementRate, likes, retweets, replies }`

**Procedure 2: `performanceIntel.patternAnalysis`**
- Input: `rangeInput`
- Fetch all published posts with latest metrics in range (reuse logic from tieredPosts or share a helper)
- Compute 3 pattern dimensions WITHOUT calling AI (pure data computation):
  1. **Format patterns**: Group by contentType (POST/THREAD/ARTICLE), compute avg engagement rate per format, identify best/worst format
  2. **Time patterns**: Group by day-of-week and hour (UTC), compute avg engagement rate per slot, identify top 3 posting windows
  3. **Topic signals**: Extract the first 5 words from each post content as a rough topic proxy, group posts by these word clusters. For top-tier posts, extract the most common 2-3 word phrases. (This is a lightweight heuristic -- not AI.)
- Return: `{ formatPatterns: [{ format, postCount, avgEngRate, avgImpressions }], timePatterns: [{ day, hour, avgEngRate, postCount }], topicSignals: [{ phrase, occurrences, avgEngRate }] }`

**Procedure 3: `performanceIntel.sparklineData`**
- Input: `z.object({ postId: z.string() })`
- Fetch all PostMetrics rows for the given post, ordered by `fetchedAt asc`
- Return: `{ postId, series: [{ timestamp: fetchedAt ISO string, impressions, engagements, engagementRate }] }`
- This enables rendering a sparkline of engagement trajectory over time per post.

Also create a batch variant: `performanceIntel.sparklineBatch`
- Input: `z.object({ postIds: z.array(z.string()).max(50) })`
- For each postId, fetch all PostMetrics rows ordered by fetchedAt asc
- Return: `Record<postId, [{ timestamp, impressions, engagements, engagementRate }]>`

**Procedure 4: `performanceIntel.insightCards`**
- Input: `rangeInput`
- First check for existing non-dismissed AIInsight rows of type `PERFORMANCE_PATTERN` generated in the last 24 hours. If found, return them.
- If stale or missing: call `generateInsight('performance_patterns', context, { model: 'claude-3-5-haiku-20241022', maxTokens: 1024 })` where context is a summary of:
  - Top 10 posts by engagement rate (content snippet, contentType, platform, engagement metrics)
  - Format pattern stats (avg engagement per contentType)
  - Best posting windows from time analysis
- Prompt Claude to return JSON array of 3-5 insight cards, each with: `{ title: string, body: string, metric: string, category: 'format'|'timing'|'topic' }`
  - Example: `{ title: "Threads outperform single posts", body: "Your threads about RWA average 3.2x more engagement than single posts", metric: "3.2x", category: "format" }`
- Persist each card as an AIInsight row (insightType: PERFORMANCE_PATTERN, content: the card JSON)
- Return: the array of insight cards

**Step 3: Register the router**
In `lib/routers/app.js`, import `performanceIntelRouter` and merge it into the appRouter under the key `performanceIntel`.
